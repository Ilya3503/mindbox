## **Deployment для веб-приложения**

Исходя из описанной в задании логики, необходимо подготовить манифест для развёртывания веб-приложения в Kubernetes. Речь идёт о stateless-приложении.

Поскольку приложение планируется предоставлять пользователям, я считаю, нужно использовать стратегию RollingUpdate. Она встроенная, для неё не нужны дополнительные инструменты, и она позволяет снизить даунтайм при обновлении версии или перезапуске приложения.

Эта стратегия при обновлении создаёт параллельный ReplicaSet и постепенно убивает старые поды, перенаправляя трафик на новые.
